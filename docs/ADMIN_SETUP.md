# 관리자 페이지 설정 가이드

## 현재 상태

### 개발 환경 (현재)
- **데이터 저장**: 브라우저 LocalStorage (임시)
- **인증**: 없음 (누구나 접근 가능)
- **백엔드**: 없음
- **용도**: 개발 및 테스트용

### 프로덕션 환경 (필요한 작업)

## 필요한 보완 사항

### 1. 인증 시스템 (필수)
```typescript
// 예시: 간단한 비밀번호 인증
- 관리자 비밀번호 설정
- 로그인 페이지 생성
- 세션 관리
```

### 2. 백엔드 API (필수)
```typescript
// API 엔드포인트 예시
POST /api/consultations - 상담 문의 저장
GET /api/consultations - 상담 문의 목록 조회
DELETE /api/consultations/:id - 상담 문의 삭제
```

### 3. 데이터베이스 연동 (필수)
- MongoDB, PostgreSQL, MySQL 등
- 상담 문의 데이터 영구 저장
- 사용자 관리

### 4. 보안 강화
- HTTPS 필수
- CSRF 보호
- Rate Limiting
- 입력 검증

## 웹 호스팅 옵션

### 옵션 1: Vercel (추천)
- **장점**: Next.js 최적화, 무료 플랜, 자동 배포
- **백엕드**: Vercel Serverless Functions
- **데이터베이스**: Vercel Postgres, MongoDB Atlas

### 옵션 2: AWS / GCP / Azure
- **장점**: 확장성, 다양한 서비스
- **단점**: 설정 복잡, 비용

### 옵션 3: Railway / Render
- **장점**: 간단한 설정, 합리적인 가격
- **데이터베이스**: 내장 또는 외부 연동

## 구현 단계

### Phase 1: 인증 시스템 추가 (우선)
1. 로그인 페이지 생성
2. 환경 변수로 비밀번호 관리
3. 세션/쿠키 기반 인증

### Phase 2: 백엔드 API 구축
1. Next.js API Routes 생성
2. 데이터베이스 스키마 설계
3. CRUD 작업 구현

### Phase 3: 프로덕션 배포
1. 환경 변수 설정
2. 데이터베이스 연결
3. 보안 설정

## 현재 사용 가능한 기능

✅ 상담 문의 폼 제출
✅ LocalStorage에 임시 저장
✅ 관리자 페이지에서 목록 확인
✅ 상세 보기 및 삭제

## 제한 사항

❌ 인증 없음 (보안 위험)
❌ 데이터 영구 저장 안됨 (브라우저 삭제 시 손실)
❌ 여러 기기에서 공유 불가
❌ 백업 없음
