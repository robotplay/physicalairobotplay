## 📋 테스트 시나리오

### 3. 카드번호 오류 테스트
- **카드번호**: `4000-0000-0000-0002`
- **예상 결과**: 결제 실패 (카드번호 오류)

## ⚠️ 중요 사항

### 테스트 모드 특징
- ✅ **실제 결제 발생 안 함**
  - 테스트 모드에서는 실제 결제가 발생하지 않습니다
  - 테스트 결제는 자동으로 취소됩니다

- ✅ **임의의 정보 사용 가능**
  - 카드번호, CVC, 비밀번호 등은 테스트용이므로 임의의 값 사용 가능
  - 단, 카드번호는 위의 테스트 카드 번호를 사용해야 함

- ✅ **유효기간은 미래 날짜**
  - 유효기간은 반드시 미래 날짜여야 합니다
  - 예: 현재가 2024년이면 `12/25` 또는 `01/26` 등

### 실운영 모드
- ⚠️ **실운영 모드에서는 실제 카드 정보 필요**
  - 테스트 카드는 실운영 모드에서 작동하지 않습니다
  - 실운영 전환 시 실제 카드 정보를 사용해야 합니다

## 🧪 테스트 방법

### 1단계: 결제 페이지 접속
```
http://localhost:3000/program/airplane
```

### 2단계: 신청서 작성 및 제출

### 3단계: 결제 버튼 클릭

### 4단계: 테스트 카드 정보 입력
- **카드번호**: `4242-4242-4242-4242`
- **유효기간**: `12/25` (또는 미래 날짜)
- **CVC**: `123`
- **카드 비밀번호**: `1234` (또는 임의의 4자리)

### 5단계: 결제 완료 확인
- 결제 성공 페이지로 리다이렉트되어야 함
- 결제 정보가 정상적으로 저장되어야 함

## 📞 문제 발생 시

### 테스트 카드가 작동하지 않는 경우
1. **카드번호 형식 확인**
   - 하이픈(-) 포함 여부 확인
   - 공백 없이 입력

2. **유효기간 확인**
   - 미래 날짜인지 확인
   - MM/YY 형식 확인

3. **브라우저 콘솔 확인**
   - 개발자 도구(F12) → Console 탭
   - 에러 메시지 확인

4. **Toss Payments 설정 확인**
   - 포트원 대시보드에서 Channel 설정 확인
   - MID가 `iamporttest_3`로 설정되어 있는지 확인
