# 소셜 로그인 설정 가이드

## 📋 개요

현재 시스템은 **하이브리드 수강신청 시스템**이 완성되어 있습니다:
- ✅ 비회원 수강 신청 (이메일 + 접근 코드)
- ✅ JWT 기반 관리자/강사 인증
- ✅ 수강 접근 토큰 시스템

소셜 로그인은 **선택적 기능**으로, 다음 단계에서 추가할 수 있습니다.

---

## 🎯 현재 시스템의 장점

### 1. 낮은 진입 장벽
- 회원가입 없이 즉시 수강 신청
- 이메일만으로 수강 가능
- 복잡한 인증 과정 불필요

### 2. 유연한 접근 방식
- 이메일 링크로 즉시 접근
- 6자리 코드로 간편 접근
- 365일 유효한 접근 토큰

### 3. 점진적 전환
- 비회원으로 시작
- 만족하면 회원 전환
- 기존 수강 내역 자동 연결

---

## 🚀 소셜 로그인 추가 시 고려사항

### Phase 3.1: 소셜 로그인 (선택적)

**장점:**
- 회원 관리 용이
- 재방문율 증가
- 맞춤형 서비스 제공

**단점:**
- 초기 진입 장벽 증가
- 개발/유지보수 비용
- OAuth 설정 복잡도

### 권장 접근 방식

```
현재 시스템 (비회원 우선)
    ↓
사용자 만족도 확인
    ↓
필요시 소셜 로그인 추가
    ↓
회원 전환 유도 (혜택 제공)
```

---

## 💡 소셜 로그인 구현 방법 (향후)

### 옵션 A: NextAuth.js (추천)

**장점:**
- Next.js 공식 지원
- 다양한 Provider 지원
- 세션 관리 자동화

**설정:**
```bash
# MongoDB 6으로 다운그레이드 필요
npm install next-auth@beta @auth/mongodb-adapter
```

**Provider:**
- Google OAuth
- Kakao OAuth
- Naver OAuth

### 옵션 B: 자체 OAuth 구현

**장점:**
- 완전한 제어
- 기존 JWT 시스템과 통합
- MongoDB 7 호환

**구현:**
1. Google/Kakao OAuth 클라이언트 생성
2. 콜백 엔드포인트 구현
3. 사용자 정보 저장
4. JWT 토큰 발급

## 🎯 권장 로드맵

### 단기 (1-2개월)
1. ✅ 현재 시스템 안정화
2. ✅ 사용자 피드백 수집
3. ✅ 수강 데이터 분석

### 중기 (3-6개월)
1. 회원 전환율 분석
2. 소셜 로그인 필요성 평가
3. OAuth 설정 준비

### 장기 (6개월 이상)
1. 소셜 로그인 구현
2. 포인트/쿠폰 시스템
3. 회원 등급제
