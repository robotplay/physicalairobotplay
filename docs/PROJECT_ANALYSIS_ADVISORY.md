## 1. Executive Summary

### 프로젝트 개요
천안 기반 로봇 코딩 교육 학원의 **통합 운영 플랫폼**으로, 학원 홍보 웹사이트와 온라인 강좌 시스템을 동시에 구축하고 있습니다.

### 현재 완성도: **75-80%** (핵심 기능 기준)

### 핵심 수치
| 항목 | 수량 |
|------|------|
| 페이지 | 25개+ |
| API 엔드포인트 | 118개 |
| React 컴포넌트 | 70개+ |
| DB 컬렉션 | 25개 |
| 관리자 탭 | 20개 |

### 종합 평가
**매우 탄탄한 기반**이 구축되어 있습니다. 학원 운영에 필요한 대부분의 백오피스 기능이 구현되어 있으며, 확장 가능한 아키텍처를 갖추고 있습니다. 다만 **온라인 강좌 실제 콘텐츠**와 **학부모/학생 사용자 경험** 부분에서 추가 개발이 필요합니다.

## 3. 강점 분석

### 3.1 기술 스택 선정 우수
```
Next.js 16 + React 19 → 최신 안정 버전
MongoDB → 유연한 데이터 모델링
Vercel → 쉬운 배포, 엣지 최적화
TipTap → 확장성 높은 에디터
```

### 3.2 확장 가능한 아키텍처
- **모듈화된 컴포넌트 구조**: 70개+ 재사용 가능한 컴포넌트
- **RESTful API 설계**: 118개 엔드포인트, 일관된 응답 형식
- **중앙화된 DB 연결**: `lib/mongodb.ts`에서 관리

### 3.3 마케팅 인프라 완비
- 랜딩 페이지 시스템
- 리드 수집/전환 추적
- 팝업/리드 매그넷
- 뉴스레터 시스템
- 뉴스 자동 수집 (차별화 요소)

### 3.4 관리자 기능 충실
- 20개 관리 탭으로 거의 모든 운영 업무 커버
- 리치 텍스트 에디터로 콘텐츠 작성 용이
- 분석/통계 기능

## 5. 온라인 강좌 전략

### 5.1 단계별 접근

#### Phase 1: 콘텐츠 인프라 구축 (우선)
```
목표: 기본적인 온라인 학습이 가능한 환경

1. 비디오 호스팅 결정
   - 옵션 A: YouTube 비공개 영상 (비용 무료, 간편)
   - 옵션 B: Vimeo Pro ($20/월, 브랜딩 제거)
   - 옵션 C: Vercel Blob + HLS (자체 호스팅, 초기 비용)

   권장: 초기 YouTube → 성장 시 Vimeo 전환

2. 강좌 구조 설계
   /online-courses/[courseId]
   ├── 개요 (설명, 커리큘럼)
   ├── 챕터 목록
   │   ├── 레슨 1 (비디오 + 설명)
   │   ├── 레슨 2
   │   └── 퀴즈
   └── 수료증

3. 진도 관리 시스템
   - 비디오 시청 기록 저장
   - 완료 표시 (체크박스 또는 자동)
   - 진도율 계산 및 표시
```

#### Phase 2: 학습 경험 강화
```
1. 퀴즈 시스템
   - 객관식/주관식 문제
   - 자동 채점
   - 오답 노트

2. 코딩 실습 환경 (선택적)
   - 웹 기반 코드 에디터 (CodeMirror/Monaco)
   - Scratch/블록 코딩 연동
   - 실행 결과 확인

3. 게이미피케이션
   - 포인트/뱃지 시스템
   - 리더보드
   - 일일 학습 스트릭
```

#### Phase 3: 수익화 강화
```
1. 다양한 가격 모델
   - 개별 강좌 구매
   - 월간/연간 구독
   - 번들 패키지

2. 무료 미리보기
   - 첫 1-2개 레슨 무료
   - 트레일러 영상

3. 할인/쿠폰 시스템
```

### 5.2 추천 온라인 강좌 콘텐츠

#### 초급 과정 (Basic Course 기반)
```
강좌 1: 로봇 코딩 첫걸음
- 대상: 코딩 입문자
- 내용: 기초 개념, 블록 코딩 소개
- 분량: 10개 레슨 (총 3시간)

강좌 2: 나의 첫 로봇 프로젝트
- 대상: 기초 학습자
- 내용: 간단한 로봇 제어
- 분량: 8개 레슨 (총 2.5시간)
```

#### 중급 과정 (Advanced Course 기반)
```
강좌 3: 센서와 로봇 제어
- 대상: 기초 완료자
- 내용: 센서 활용, 조건문, 반복문
- 분량: 12개 레슨 (총 4시간)

강좌 4: 자율주행 로봇 만들기
- 대상: 중급 학습자
- 내용: 라인트레이싱, 장애물 회피
- 분량: 15개 레슨 (총 5시간)
```

#### 드론 과정 (AirRobot Course 기반)
```
강좌 5: 드론 비행의 기초
- 대상: 드론 입문자
- 내용: 드론 원리, 기본 조종
- 분량: 10개 레슨 (총 3시간)

강좌 6: 드론 코딩과 미션 수행
- 대상: 드론 기초 완료자
- 내용: 프로그래밍 비행, 미션 수행
- 분량: 12개 레슨 (총 4시간)
```

### 5.3 차별화 전략

```
3. 프로젝트 중심
   - 매 강좌 완성 프로젝트 제공
   - 프로젝트 갤러리 공유
   - 우수작 시상
```

## 6. 학원 홍보 전략

### 6.1 현재 홍보 인프라 활용

#### A. 뉴스 자동 수집 시스템 (구축 완료)
```
활용 방안:
1. 로봇/AI 관련 최신 뉴스 자동 큐레이션
2. 부모님들에게 "왜 로봇 교육이 중요한지" 근거 제공
3. SNS 공유용 콘텐츠 소스

개선 제안:
- 뉴스에 학원 관점의 코멘트 추가
- 관련 수업/프로그램 연결 링크
```

#### B. 랜딩 페이지 최적화
```
현재 3개 랜딩:
1. /landing/free-trial → 무료 체험
2. /landing/summer-camp → 여름 캠프
3. /landing/competition-prep → 대회 준비

추가 제안:
4. /landing/first-coding → 첫 코딩 체험
5. /landing/winter-camp → 겨울 캠프
6. /landing/parent-guide → 학부모 가이드
```

#### C. 팝업 시스템 활용
```
시나리오별 팝업:
- 첫 방문 → 무료 체험 안내
- 30초 이상 체류 → 상담 신청 유도
- 이탈 의도 감지 → 특별 할인 제안
- 특정 페이지 → 관련 이벤트 안내
```

### 6.2 콘텐츠 마케팅 전략

#### A. 블로그/뉴스 콘텐츠
```
주간 콘텐츠 계획:
- 월: 로봇 교육 트렌드/뉴스
- 수: 학생 프로젝트 소개
- 금: 학부모 교육 정보 (진로, 대회 등)

SEO 키워드:
- 천안 로봇 학원
- 어린이 코딩 교육
- 로봇 대회 준비
- 드론 교육
- 피지컬 컴퓨팅
```

#### B. 성공 사례 강화
```
콘텐츠:
- 대회 수상 스토리
- 학생 인터뷰
- Before/After 성장기
- 학부모 후기

형식:
- 텍스트 + 사진
- 인터뷰 영상 (짧은 클립)
- 인포그래픽
```

### 6.3 소셜 미디어 전략

#### A. 플랫폼별 전략
```
Instagram:
- 수업 현장 사진/영상
- 학생 작품 쇼케이스
- 스토리: 일상/비하인드

- 이벤트 알림
```

#### B. 콘텐츠 재활용
```
하나의 콘텐츠 → 여러 플랫폼:
1. 학생 프로젝트 촬영
2. → Instagram 릴스 (30초)
3. → YouTube Shorts (60초)
4. → 블로그 상세 글
5. → 웹사이트 갤러리
```
